# Q80 — Stage 3 hypotheses (fixed before computation)

Four hypotheses on whether the outreach verdict survives asynchronous update, where one element acts each
step and the others hold (a stochastic mixture, probability 1/n per element). Written and committed before
any test runs.

## H1 — The read-recipient triad stays triadic

- **Claim:** Under asynchronous update the read-recipient form keeps Φ > 0 (triadic).
- **H0:** The asynchronous read-recipient form is dyadic.
- **Predicted outcome:** triadic. H0 refuted.

## H2 — The broadcast stays dyadic

- **Claim:** Under asynchronous update the broadcast keeps Φ ~ 0 (dyadic); asynchrony does not manufacture
  a triad.
- **H0:** The asynchronous broadcast is triadic.
- **Predicted outcome:** dyadic. H0 refuted.

## H3 — The all-required campaign stays triadic

- **Claim:** Under asynchronous update the all-required campaign keeps Φ > 0 (triadic).
- **H0:** The asynchronous all-required campaign is dyadic.
- **Predicted outcome:** triadic. H0 refuted.

## H4 — The verdict is preserved for every keystone

- **Claim:** Asynchronous update preserves the synchronous dyadic/triadic verdict for all three keystones;
  synchronicity is not load-bearing for the verdict.
- **H0:** Some keystone's verdict flips under asynchronous update.
- **Predicted outcome:** every verdict preserved. H0 refuted.
