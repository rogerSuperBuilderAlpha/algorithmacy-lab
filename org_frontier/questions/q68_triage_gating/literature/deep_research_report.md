# Q68 — Stage 2 deep research: filtering, triage, and gatekeeping

This study draws on the same literatures as Q63-Q67 (shared `references.bib`) and the receiving side of
the channel.

## Filtering on the receiving side

The economics of spam centres on the receiving side's defences: filters and triagers that decide which
messages reach attention [rao2012economics]. A triage agent is an automated gatekeeper between sender and
recipient. AI-mediated communication places agents on both sides of an exchange [hancock2020aimc;
jakesch2019aimc], so the recipient's agent triaging a sender's agent's message is the two-sided case.

## Gatekeeping and coordination

Coordination theory treats a gatekeeping mechanism as part of the coordination structure when it
constrains the flow [malone1994interdisciplinary]. Whether the gatekeeper is part of the irreducible
coordination, rather than an external filter, is the structural question. Integrated information theory
answers it through major-complex membership [oizumi2014phenomenology; albantakis2023iit4; mayner2018pyphi]:
an element belongs to the core only if it is bound bidirectionally, which the lab's principal/gating
finding made precise.

## The open gap

The signaling and spam literatures describe triage as a filter; none computes whether the triage agent is
inside the irreducible coordination. The gap is open: build the triage agent under gating-only,
monitoring-only, and bidirectional coupling, and read its core membership. In-silico, with the validation
gap [axtell1996aligning]; four hypotheses fixed before computing [chamberlin1965method; platt1964strong;
brodeur2024preregistration].
